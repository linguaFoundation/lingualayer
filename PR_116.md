# feat(license-router): cross-contract QualityOracle royalty multiplier

Closes #116

## Summary

`LicenseRouter` now reads `QualityOracle.royalty_multiplier_bps` via a
cross-contract call and applies the quality-tier multiplier before
transferring royalty payments. A base amount supplied by the caller is
scaled by the dataset's attested quality tier, then transferred to the
recipient via a SEP-41 `token::Client`.

**Multiplier table**

| Tier     | BPS    | Effective rate |
|----------|--------|----------------|
| Unrated  | 10 000 | 1.0×           |
| Bronze   |  7 500 | 0.75×          |
| Silver   | 10 000 | 1.0×           |
| Gold     | 12 500 | 1.25×          |
| Platinum | 15 000 | 1.5×           |

Formula: `adjusted_amount = base_amount × multiplier_bps / 10_000`

## Changes

### `contracts/license-router`

- **`initialize(admin, oracle)`** — stores the `QualityOracle` contract
  address alongside the admin at setup time.
- **`route_license(caller, dataset_id, base_amount, token, recipient) → LicenseRouteResult`**
  — fetches the multiplier cross-contract, scales the base amount, and
  executes the token transfer.
- **`oracle_address()`** — read-only view returning the bound oracle address.
- Version bumped **1 → 2**.
- New `LicenseRouteResult` type exposes `base_amount`, `multiplier_bps`,
  and `adjusted_amount` for transparency to callers.
- `Cargo.toml` — promoted to `cdylib + rlib`, added `testutils` feature
  and `quality-oracle` path dev-dependency.

### `contracts/quality-oracle`

- Replaced all `format!`-based storage keys with typed `StorageKey` enum
  variants (`Quality(String)`, `Attestation(String, Address)`,
  `Curator(Address)`). `format!` is unavailable in `#![no_std]` WASM
  contexts; this was a latent bug that would have caused compile failures
  on the WASM target.
- Fixed unclosed `if` block in `attest_quality` left by the slashing PR merge.
- Preserved all slashing logic (`slash_curator`, `get_curator_stake`,
  `min_stake` enforcement) introduced in PR #162.
- `Cargo.toml` — promoted to `cdylib + rlib` with `testutils` feature gate
  so the crate can be imported as a dev-dependency by `license-router`.

### Workspace

- **`rust-toolchain.toml`** (new) — pins toolchain to Rust `1.91.0` with
  `wasm32v1-none` + `wasm32-unknown-unknown` targets. `soroban-sdk 25.x`
  requires Rust ≥ 1.91 and the `wasm32v1-none` target.
- **`Cargo.toml`** — upgrades workspace `soroban-sdk` from `21.0.0` to
  `25.0.0` and deduplicates the `resolver = "2"` key.
